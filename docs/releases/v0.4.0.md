# v0.4.0 — mmx-claude (Claude Code MCP)

Adds `@hl-plugins/mmx-claude`, a sibling of `@hl-plugins/mmx` that ships
the same seven MiniMax multimodal tools to **Claude Code** via an MCP
server.

## Highlights

- **New plugin package** `packages/plugin-mmx-claude/`. Bun-bundled MCP
  server, ~70-line `runMmx` + path/suspicious-path helpers, two bun
  test suites.
- **Two new contract fields** on the `hl-plugins` manifest:
  `claudeMcp` (bundled server) and `claudeSkill` (markdown skill). A
  plugin can target OpenCode, Claude Code, or both — any subset of the
  four contract fields is allowed.
- **Install flow extended** to copy the MCP bundle to
  `~/.hmanlab/plugins/<plugin>/`, copy the skill to
  `~/.claude/skills/<plugin>/SKILL.md`, and register
  `mcpServers[name] = { command: "bun", args: [<abs path>] }` in
  `~/.claude.json`. Idempotent and symmetric with the OpenCode path.
- **New install hint**: when `~/.claude/` exists but `~/.opencode/`
  does not, and the user is installing `mmx`, prints a yellow heads-up
  about `mmx-claude` and proceeds.
- **Defensive `~/.claude.json` parsing**: empty file → `{}`; non-object
  top-level → clear error message; other top-level keys preserved.
- **52 new tests** (29 node + 23 bun). Typecheck, format, build all
  green.

## Unchanged

- `packages/plugin-mmx/` source — zero edits. The OpenCode plugin
  stays as-is.

## Out of scope (deferred)

- Slash commands (`/mmx-image ...`) — could ship in a v2 alongside
  the MCP server.
- Auto-update of the MCP bundle — `hl-plugins update` needs a
  Claude-side mirror.
- Cross-agent state sync — both agents already share `mmx-cli` state.
- Publish to npm — both packages stay `private: true` until the
  install flow is battle-tested on real Claude Code installs.
